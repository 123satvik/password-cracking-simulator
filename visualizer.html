Big-O Summary
BRUTE
O(|Σ|ⁿ) — exponential in length n
DICT
O(W × M) — linear in words × mutations
RAINBOW
O(1) lookup (space-time tradeoff)
SALTED
Defeats rainbow tables → O(|Σ|ⁿ)
BCRYPT
Designed slow — 1000× harder than SHA
Defence Recommendations
✓ Use Argon2id / bcrypt / scrypt (slow hashes)
✓ Minimum 12 characters
✓ Mix uppercase, lowercase, digits, symbols
✓ Always use unique salts per password
✓ Implement rate limiting + lockout
✗ Never store plaintext passwords
✗ Avoid MD5/SHA-1 for password storage
